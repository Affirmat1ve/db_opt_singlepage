УРОВЕНЬ 1: СРЕДСТВА МОНИТОРИНГА ОПЕРАЦИОННОЙ СИСТЕМЫ

1.1 Описание категории инструментов

Средства мониторинга операционной системы — это базовый уровень наблюдения за работой ClickHouse. Поскольку ClickHouse работает на Linux и интенсивно использует системные ресурсы (CPU, память, дисковый ввод-вывод), мониторинг на уровне ОС критически важен для:

  • Выявления узких мест производительности
  • Понимания потребления ресурсов процессом clickhouse-server
  • Диагностики проблем с оборудованием
  • Планирования масштабирования

1.2 Основные инструменты

Инструмент Назначение Ключевые метрики
top/htop Мониторинг процессов и CPU %CPU, %MEM, RES, TIME+
vmstat Виртуальная память, CPU r, b, swpd, free, bi, bo, wa
iostat Дисковый I/O r/s, w/s, rkB/s, wkB/s, %util, await
iotop I/O по процессам DISK READ, DISK WRITE, IO%

2. ПРАКТИЧЕСКИЙ БЛОК

2.1 Установка и запуск ClickHouse

Шаг 1: Скачивание

curl https://clickhouse.com/ | sh

Шаг 2: Установка как сервис

sudo ./clickhouse install

Шаг 3: Запуск сервера

sudo clickhouse start

Шаг 4: Проверка подключения

clickhouse-client

Шаг 5: Теперь можно писать запросы напрямую после :)

SELECT 1

Теперь работа будет устроена следующим образом: оставьте это окно, откройте новое. Для нового окна: 1) Запустите указанную утилиту. Посмотрите на показатели при фоновом процессе. 2) Вернитесь к окну 1. с открытым clickhouse-client и запустите в нем запрос:

SELECT
count(),
avg(number),
max(number)
FROM numbers(10000000);

3) В этот же момент смотрите на окно 2. со статистиками и то, как они меняются.

2. ВИЗУАЛЬНЫЙ БЛОК

2.1 Примеры вывода команд

A. top — мониторинг процесса ClickHouse

Верхняя часть: статистика системы

Колонка / Поле Описание Норма для ClickHouse
load average (1, 5, 15 мин) Средняя загрузка системы: количество процессов в очереди на CPU < числа ядер CPU (для 8 ядер: < 8.0)
Tasks: running Процессы, активно выполняющиеся или готовые к выполнению 1-5 (без нагрузки)
Tasks: sleeping Процессы в режиме ожидания Любое значение, норма
Tasks: zombie Зомби-процессы (завершённые, но не удалённые) Должно быть 0
%Cpu(s): us (user) Время CPU в пользовательском режиме (приложения) < 70% (остальное — запас)
%Cpu(s): sy (system) Время CPU в режиме ядра (системные вызовы) < 20%
%Cpu(s): ni (nice) Время для процессов с изменённым приоритетом Обычно 0%
%Cpu(s): id (idle) Время простоя CPU > 10% (есть запас производительности)
%Cpu(s): wa (iowait) Критично! Время ожидания операций с диском < 10% для SSD, < 20% для HDD
%Cpu(s): hi/si Время на обработку прерываний (hardware/software) < 5%
%Cpu(s): st (steal) Время, «украденное» гипервизором (для виртуальных машин) 0% (на физических серверах)
MiB Mem: free Свободная оперативная память > 20% от общей
MiB Mem: buff/cache Память под буферы и кэш файловой системы Может быть большой — ClickHouse использует для кэша
MiB Mem: avail Доступная память для новых приложений > 10-15% от общей
MiB Swap: used Использованный swap Должно быть 0! Свопинг критичен для БД

Таблица процессов

Колонка Описание Норма для ClickHouse
PID Уникальный идентификатор процесса Постоянный для clickhouse-server
USER Пользователь, от имени которого запущен процесс Обычно clickhouse
PR (Priority) Приоритет процесса (меньше = выше приоритет) Обычно 20
NI (Nice) «Вежливость» процесса (-20...+19) Обычно 0
VIRT Виртуальная память процесса (включая swap, отображённые файлы) Может быть большой — не показатель реальной нагрузки
RES Важно! Физическая память (RAM), которую процесс фактически использует Основной показатель потребления памяти; должен помещаться в RAM
SHR Разделяемая память (библиотеки, IPC) Вспомогательная метрика
S (Status) Статус: R=running, S=sleeping, D=disk wait, Z=zombie Обычно S (sleeping) или R (при нагрузке)
%CPU Процент использования CPU процессом Может быть >100%! 100% = одно ядро; ClickHouse многопоточный
%MEM Процент использования оперативной памяти (RES / total RAM) 10-40% в зависимости от объёма RAM и нагрузки
TIME+ Общее процессорное время с момента запуска Растёт со временем; показывает накопленную нагрузку
COMMAND Имя процесса clickhouse-server (сервер) или clickhouse-client (клиент)

B. vmstat — комплексный мониторинг

Запустите во втором окне vmstat. Для этого введите:

vmstat 2
Группа Колонка Описание Норма для ClickHouse
procs r (runnable) Процессы, готовые к выполнению и ожидающие CPU < числа ядер CPU
procs b (blocked) Процессы в uninterruptible sleep (ожидают I/O) Должно быть 0 (иначе проблемы с диском)
memory swpd Использовано swap-памяти (в KiB) Должно быть 0! Свопинг недопустим для БД
memory free Свободная оперативная память (в KiB) > 20% от общей
memory buff Память под буферы ядра (в KiB) Может быть большой — норма
memory cache Память под кэш файловой системы (в KiB) ClickHouse использует для кэширования данных
swap si (swap in) Чтение из swap в RAM (KiB/сек) Должно быть 0!
swap so (swap out) Запись из RAM в swap (KiB/сек) Должно быть 0!
io bi (blocks in) Блоки, прочитанные с диска в секунду Зависит от нагрузки; растёт при SELECT
io bo (blocks out) Блоки, записанные на диск в секунду Зависит от нагрузки; растёт при INSERT
system in (interrupts) Прерываний в секунду (аппаратных + программных) Зависит от активности сети/диска
system cs (context switches) Переключений контекста в секунду < 10 000 (выше — возможна перегрузка CPU)
cpu us (user) % времени CPU в пользовательском режиме < 70%
cpu sy (system) % времени CPU в режиме ядра < 20%
cpu id (idle) % времени простоя CPU > 10% (есть запас)
cpu wa (iowait) Критично! % времени ожидания операций с диском < 10% для SSD, < 20% для HDD
cpu st (steal) Время, украденное гипервизором (для виртуальных машин) 0% (на физических серверах)

sudo iotop

sudo iotop -o
